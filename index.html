專為NMON設計的維運平台

NMON是由IBM免費提供且opensource 監控工具，主要用來監控和分析AIX/Linux系統資源使用情況。
BiMAP NMON 自動化分析平台，就是利用定時收集NMON 的資訊，自動產生分析報表，並且提供專業的諮詢服務，以確保主機高速穩定運行。

BiMAP-NMON 常見問題集

 BiMAP-NMON 可以支持哪些nmon的版本 

可以支持所有的 AIX和Linux 上產生的nmon 格式和不同版本的log

 BiMAP-NMON可以同時分析幾台nmon的log

NMONs 大數據分析平台在一個用戶帳號中，可以同時分析128臺不同server的nmon log，並且依據不同的核心系統進行分類和關聯分析，如不同DB server，AP server 之間進行對比分析

 BiMAP-NMON可以結合哪些log一起關聯分析

目前除了AIX/Linux nmon 日誌外，更可以結合其他的日誌，一起關聯分析：
* Microsoft Windows LOGMAN 計數器日誌
* Oracle AWR/stackpack Report 日誌
* 還可以支持各種不同存儲設備的log(如EMC、HDS、IBM、HP、netapp等)

 如何上傳主機上的nmon log

在申請用戶帳號後，即可使用2種上傳的模式，
* 包含通過web 手動上傳nmon 歷史log，
* 或是經由ftp的方式，設定每天自動上傳昨天收集的nmon log;
* 發送的格式可以為原始的nmon格式或是經由壓縮的rar/zip/7z/tar.gz 的壓縮格式

 定期收集NMON 日誌參數設定建議

為了更精準的進行大數據分析，在設定nmon 的收集命令時， 建議收集全天24小時、採樣點為1440次，即60秒的收集間隔，并增加Top Process的運行記錄
* 建議如下： nmon12e_aix61 -F server_201503010000.nmon -fT -s 60 -c 1440
* 收集過程占用CPU負載率小於0.5%，不會影響系統的正常運行。

 BiMAP-NMON 如何做趨勢預測分析

BiMAP-NMON 支援5年以上的歷史資料，從這些歷史log 中，進行數據挖掘、統計、篩選、歸納，以建立系統負載趨勢統計，以當成系統升級改造方案的量化依據，避免設備採購後，改善效果不如預期的風險。

通過NMON 收集的系統日誌， 包含CPU、Memory、Disk、Network、Filesystem、PID等，並結合Oracle AWR 、Microsoft LOGMON 、Storage 等系統日誌，進行關聯分析和交叉比對，以達到效能監控、效能分析和效能預測的功效，以降低營運成本和提供生產效率的目標。
